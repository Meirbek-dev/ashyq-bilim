# ShadCN Migration + Theming Refactor

**Systematically refactor and rewrite existing components and pages** to use the **shadcn design system**, while introducing a **robust, scalable theming architecture**.

---

## Objectives

1. Replace custom / inconsistent UI with **shadcn/ui components**
2. Standardize styling using **Tailwind + design tokens**
3. Implement **first-class theming (light/dark + extensible themes)**
4. Improve **accessibility, composition, and consistency**
5. Preserve **existing functionality and business logic**

---

## Constraints

* Do NOT break existing behavior
* Prefer **composition over inheritance**
* Avoid inline styles unless absolutely necessary
* Follow **App Router + Server/Client component boundaries correctly**
* Do NOT introduce unnecessary dependencies
